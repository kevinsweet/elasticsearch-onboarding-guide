Conversation Playbook

The playbook defines a conversation flow — ingest-first by design. Get data in with minimal questions, then ask about intent once the actual data shape is visible. Recommendations are better when informed by real fields.

WHY THIS ORDER
Ingest is the first action. The only question before it is "where's your data?" — the minimum needed to connect. Intent questions ("what are you building?", "who searches?") come after data is in, because the agent can see the actual fields and make informed recommendations instead of asking abstract architecture questions.
One question at a time. Each answer narrows the path. The agent skips steps that don't apply. A keyword-only user never sees embedding questions. A BYO-vector user never picks a model. The order is relative — steps can compress, expand, or reorder based on what the user already has.

Two-Pass Ingest (Step 2 → Step 4)

A proper mapping can't be recommended without seeing the data, and the data can't be seen without loading it first. The user experiences this as "data went in, then search got better when configured properly."

Pass 1: Quick ingest

Auto-mapped. Get data visible fast.

  • Bulk load from source
  • Auto-detected types
  • No manual mapping yet
  • "10 products indexed"
→

Mapping decision

Based on intent + actual data.

  • Which fields get semantic_text?
  • Which are keyword (facets)?
  • Quantization strategy?
  • Data cleanup needed?
→

Pass 2: Final ingest

Proper index with confirmed mapping.

  • Create index with mapping
  • Reload data from source
  • Generate embeddings
  • Alias swap

Conversation Engine

The playbook sits at the center. It orchestrates when to load reference files, when to ask questions, and when to execute. Each step maps to specific reference knowledge.

Principles:
  • 1One question at a time
  • 2Ingest before asking intent
  • 3Terse prompts (no jargon)
  • 4Ask before executing (write confirmation)
  • 5Skip irrelevant steps automatically
  • 6Contextualize every step (why asking)
  • 7Save code to notebook for reference
