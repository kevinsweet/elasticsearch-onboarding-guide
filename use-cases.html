What can you build?

Four broad categories of things people build with Elasticsearch. Each breaks down into specific patterns — any of these can be the starting point for onboarding.

Search UI for humans
People type queries (or browse), see ranked results. The end user is a human interacting with a search bar, filters, or catalog page.
E-commerce product search
Search by name, filter by brand/price/category, sort by relevance or popularity. Autocomplete. Faceted navigation.
Site search / docs search
Search across a website or documentation. Highlight matching text. Rank by freshness or relevance.
Internal knowledge base
Employees search company wikis, policies, runbooks. Often combined with access control.
Marketplace / listing search
Jobs, real estate, rentals, freelancers. Geo-aware, multi-faceted, often with saved searches and alerts.
Media / content discovery
Search videos, articles, podcasts, courses. Metadata-rich, often with personalized ranking.
Log & Event Search
Searching through logs, events, traces, or telemetry data. The data looks like observability, but the interaction is search — people query, filter, and drill into results to find what happened.
Application log search
Developers searching error logs, stack traces, request logs. Filter by service, time range, severity. Find the needle in terabytes.
Security event investigation
Analysts searching auth logs, firewall events, DNS queries. Correlate across sources, filter by IP/user/time window.
Audit trail search
Compliance teams searching who did what and when. Access logs, change records, approval flows. Often needs exact-match and time-range queries.
IoT / sensor event search
Search device telemetry, alert history, state changes. Filter by device ID, event type, threshold breaches.
Transaction / request tracing
Find a specific request across distributed services. Search by trace ID, user ID, error code. Reconstruct what happened end-to-end.
Infrastructure & utility metrics search
Search large-scale time-series metrics from distributed systems — water consumption across thousands of meters, power grid readings, fleet telemetry. Filter by region, threshold, time window. Find anomalies in millions of data points.
Recs / Similarity
"Find things like this one." The system takes an item (or a user profile) and returns similar items. No explicit query from the user — the input is an embedding, not typed text.
Product recommendations
"Customers who viewed this also liked..." Embed products, find nearest neighbors to the one being viewed.
Content you might like
Articles, videos, songs similar to what you've consumed. User preference vector → find nearest content vectors.
Duplicate / near-duplicate detection
Find copies, plagiarism, similar support tickets, duplicate listings. Embed → threshold on cosine similarity.
Resume ↔ job matching
Embed resumes and job descriptions into the same space. Find best matches in either direction.
Image / visual similarity
"Find visually similar products." Embed images (CLIP, etc.), retrieve by visual similarity. Reverse image search.
RAG / Chatbot / Q&A
Retrieve relevant context from your data and feed it to an LLM to generate grounded answers. The end user asks a question in natural language; the system retrieves, then generates.
Customer support chatbot
User asks a question, system retrieves relevant docs/FAQs, LLM generates an answer citing sources.
Internal Q&A over docs
"What's our refund policy?" — retrieves from policy docs, generates a direct answer with citations.
Code assistant / repo Q&A
Ask questions about a codebase. Embed code + docs, retrieve relevant chunks, LLM explains or generates code.
Research assistant
Ask questions across papers, reports, transcripts. Retrieves relevant passages, synthesizes an answer.
Conversational product advisor
"I need a laptop for video editing under $2000" — retrieves matching products, LLM recommends with reasoning.
Agent Memory & Retrieval
Long-term context for autonomous AI agents. The agent stores facts, conversation history, and knowledge, then retrieves by meaning when it needs context for a decision.
Conversation memory
Agent remembers past interactions across sessions. Retrieves relevant history when the user references something from before.
Tool output storage
Agent runs tools (APIs, calculations, web searches), stores results for later retrieval without re-running.
Knowledge accumulation
Agent learns facts over time (user preferences, project context, domain rules). Retrieves relevant facts per task.
Multi-agent shared memory
Multiple agents share a knowledge store. One agent's findings are retrievable by others.
Workflow state persistence
Long-running agent workflows store intermediate state. Resume from where you left off, retrieve relevant context for next step.
These categories aren't mutually exclusive — many real systems combine them. An e-commerce site might have product search (Search UI) + "similar items" (Recs) + a chatbot advisor (RAG). The use case determines which retrieval pattern dominates and which fields matter most.
